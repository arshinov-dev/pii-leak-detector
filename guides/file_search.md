# Сканер файлов `file_search.py`

## Назначение

`file_search.py` отвечает за первичную инвентаризацию файлового хранилища.

Он рекурсивно обходит директорию, определяет предполагаемый формат каждого файла и возвращает структурированный результат для следующих этапов пайплайна.

Сканер не изменяет файлы, не извлекает из них персональные данные и не принимает решение об утечке. 

Его задача - дать надежное описание файла и подсказать, каким обработчиком его лучше передать дальше.

## Как определяется формат

Сканер использует несколько уровней проверки:

1. Бинарные сигнатуры для известных форматов, например Parquet, старые Office-файлы в OLE-контейнере и OOXML-файлы в ZIP-контейнере.

2. Библиотеку `filetype` для распространенных бинарных форматов: PDF, изображения, видео, архивы и часть офисных документов.

3. Анализ текстовой структуры для HTML, JSON и CSV/TSV.

4. Расширение файла как fallback, если сигнатура или структура не дали уверенного ответа.

Важно: поле `extension` означает не исходное расширение имени файла, а предполагаемый формат, который сканер вывел по доступным признакам. 

Если нужно посмотреть, как файл был назван изначально, используется `original_extension`.

## Основные поля результата

| Поле | Что означает | Пример | Зачем нужно |
|------|--------------|--------|-------------|
| `path` | Полный или переданный путь к файлу | `"share/data.csv"` | Позволяет следующему этапу открыть файл |
| `name` | Имя файла без пути | `"data.csv"` | Удобно для отчетов и логов |
| `decoded_name` | Имя файла после URL-декодирования | `"ГОСТ.docx"` | Делает `%D0%...` имена читаемыми |
| `size_bytes` | Размер файла в байтах | `650666` | Полезно для лимитов и приоритизации |
| `original_extension` | Расширение из имени файла, без точки и в нижнем регистре | `"xls"` | Показывает, как файл был назван |
| `extension` | Предполагаемый формат по сигнатуре, структуре или fallback-логике | `"html"` | Выбирает конкретный обработчик |
| `mime` | MIME-тип предполагаемого формата | `"text/html"` | Унифицированное описание формата |
| `family` | Семейство форматов | `"structured"`, `"document"`, `"image"` | Выбирает общий класс обработчика |
| `is_binary` | Является ли формат бинарным | `true` | Помогает не читать бинарные файлы как текст |
| `confidence` | Уверенность определения формата от `0.0` до `1.0` | `0.95` | Помогает отделить надежные результаты от сомнительных |
| `status` | Статус определения: `success`, `warning`, `unknown`, `error` | `"success"` | Определяет дальнейшую ветку обработки |
| `method` | Метод, которым определен формат | `"binary_magic"`, `"csv_sniffer"` | Объясняет происхождение решения |
| `message` | Краткое человекочитаемое пояснение | `"JSON успешно разобран."` | Удобно для логов и ручной проверки |
| `details` | Дополнительные детали определения | `{"encoding": "utf-8-sig"}` | Передает технический контекст downstream-блокам |

## Статусы определения

| Статус | Что значит | Рекомендуемое действие |
|--------|------------|------------------------|
| `success` | Формат уверенно определен по сигнатуре, структуре или надежному fallback | Передавать файл в следующий обработчик |
| `warning` | Есть признаки формата, но уверенность неполная | Обрабатывать осторожно и писать в технический лог |
| `unknown` | Формат не удалось надежно распознать | Применить общий fallback или отложить на ручную проверку |
| `error` | Файл не найден, недоступен или произошла ошибка чтения | Пропустить файл и сохранить ошибку в лог |

Эти статусы относятся только к качеству определения формата. 

**Они не являются оценкой риска утечки ПДн.**

## Семейства форматов

Поле `family` нужно для маршрутизации файла в следующий блок пайплайна:

| Семейство | Примеры форматов | Возможный следующий обработчик |
|-----------|------------------|--------------------------------|
| `structured` | CSV, TSV, JSON, Parquet | Парсер таблиц или структурированных данных |
| `document` | PDF, DOC, DOCX, RTF | Извлечение текста из документов |
| `spreadsheet` | XLS, XLSX | Парсер электронных таблиц |
| `presentation` | PPT, PPTX | Извлечение текста и OCR изображений на слайдах |
| `image` | JPG, PNG, GIF, TIFF, WebP | OCR и анализ изображений |
| `video` | MP4 | Извлечение кадров и OCR при необходимости |
| `web` | HTML | HTML-парсер и очистка текста |
| `text` | TXT, Markdown | Текстовый обработчик |
| `archive` | ZIP | Распаковка или отдельная политика обработки |
| `executable` | ELF | Обычно исключить из ПДн-пайплайна |

## Пример результата

Файл называется `report.xls`, но внутри содержит HTML-разметку:

```json
{
  "path": "share/report.xls",
  "name": "report.xls",
  "decoded_name": "report.xls",
  "size_bytes": 993,
  "original_extension": "xls",
  "status": "success",
  "method": "html_heuristic",
  "mime": "text/html",
  "extension": "html",
  "family": "web",
  "is_binary": false,
  "confidence": 0.9,
  "message": "HTML определен по тегам в начале файла.",
  "details": {
    "encoding": "utf-8-sig"
  }
}
```

## Рекомендуемая интерпретация `confidence`

- `confidence >= 0.8` - формат можно считать достаточно надежным для автоматической обработки.

- `0.5 <= confidence < 0.8` - формат можно обрабатывать, но стоит сохранить предупреждение в технический лог.

- `confidence < 0.5` - формат сомнительный; лучше использовать fallback или отправить на ручную проверку.

**Не является оценкой риска утечки ПДн.**

## Публичные функции

### `detect_file_type(filepath)`

Определяет формат одного файла и возвращает словарь с полями результата.

Функция не печатает прогресс и подходит для использования внутри других модулей.

### `traverse_data_folder(folder_name, verbose=True)`

Рекурсивно обходит директорию и лениво возвращает пути ко всем обычным файлам.

Поддерживаются абсолютные пути, пути относительно текущей директории запуска и папки рядом со скриптом.

### `scan_files(file_paths)`

Принимает поток путей и лениво возвращает результаты `detect_file_type` для каждого файла.

### `count_and_report_files(file_paths)`

Сканирует файлы, печатает краткую сводку по форматам и возвращает полный список результатов. 

## Как использовать в пайплайне

Пример маршрутизации:

```python
import file_search as fs

files = fs.traverse_data_folder("share", verbose=False)

for item in fs.scan_files(files):
    if item["status"] == "error":
        continue

    if item["family"] == "structured":
        # Передать в парсер CSV/JSON/Parquet.
        pass
    elif item["family"] == "image":
        # Передать в OCR.
        pass
    elif item["family"] in {"document", "spreadsheet", "presentation", "web", "text"}:
        # Передать в извлечение текста.
        pass
```
