# Планировщик извлечения `extraction_planner.py`

## Назначение

`extraction_planner.py` принимает структурированные результаты `file_search.py` и решает,
каким способом дальше извлекать текст или таблицы из каждого файла. Лимиты и keyword-списки
планировщика берутся из `detector_settings.json -> planner`.

Планировщик не читает содержимое файла, не запускает OCR, не ищет ПДн и не оценивает риск
утечки. Его задача - построить маршрут обработки по уже известным метаданным:
формат, семейство, MIME, размер, статус сканирования, имя и путь.

## Основные объекты

### `ExtractionStep`

Описывает один будущий шаг извлечения:

| Поле | Что означает |
|---|---|
| `stage` | Этап: `primary` или `escalation`. |
| `extractor` | Имя будущего извлекателя, например `pdf_text_extractor`. |
| `source` | Логический источник: `pages`, `table`, `image`, `slides`. |
| `reason` | Почему выбран этот шаг. |
| `priority` | Относительный приоритет выполнения. Меньше - раньше/важнее. |
| `optional` | Можно ли пропустить шаг, если триггер не сработал. |
| `params` | Параметры для будущего извлекателя. |

### `ExtractionPlan`

Описывает весь маршрут для одного файла:

| Поле | Что означает |
|---|---|
| `path`, `name` | Файл из результата сканера. |
| `extension`, `family` | Предполагаемый формат и семейство. |
| `status`, `confidence` | Статус и уверенность сканера. |
| `strategy` | Крупная стратегия обработки. |
| `primary_steps` | Дешевые или обязательные шаги извлечения. |
| `escalation_steps` | Дорогие или условные шаги, включая OCR. |
| `warnings` | Предупреждения маршрутизации. |
| `skip_reason` | Причина пропуска, если файл не обрабатывается. |
| `metadata` | Дополнительные признаки маршрутизации. |

## Публичные функции

### `build_extraction_plan(scan_result)`

Строит план для одного результата `file_search.detect_file_type()`.

### `plan_extractions(scan_results)`

Лениво строит планы для потока результатов `file_search.scan_files()`.

### `summarize_plans(plans)`

Возвращает счетчики по стратегиям, извлекателям и OCR-эскалациям.

Важно: счетчик `ocr_escalation_plans` означает "OCR может понадобиться после
дешевого извлечения или prefilter", а не "OCR будет запущен для всех этих файлов".

### `print_plan_report(plans)`

Печатает краткую сводку для CLI.

## Стратегии

| Стратегия | Когда используется |
|---|---|
| `structured_parse` | CSV, TSV, JSON, Parquet. |
| `cheap_text` | HTML, TXT, Markdown, RTF. |
| `table_parse` | XLS/XLSX. |
| `pdf_text_then_targeted_ocr` | PDF: сначала цифровой текст, затем OCR только для нужных страниц. |
| `docx_text_then_embedded_ocr` | DOCX: текст, таблицы, headers/footers, затем OCR крупных картинок. |
| `office_conversion` | Старые DOC-файлы через специализированный extractor или LibreOffice. |
| `legacy_office_container` | OLE-контейнер с неочевидным расширением. |
| `presentation_text_then_embedded_ocr` | PPT/PPTX: текст слайдов и условный OCR картинок. |
| `image_prefilter_then_ocr` | Изображение: сначала prefilter, затем OCR для скана/документа/скриншота. |
| `video_sparse_frame_ocr` | MP4: метаданные и редкий OCR кадров с лимитами. |
| `defer_archive` | Архивы требуют отдельной политики безопасной распаковки. |
| `skip` | Пустые, исполняемые или бесполезно маленькие файлы. |

## OCR-политика

OCR в плане почти всегда находится в `escalation_steps`.

Это означает не "запустить OCR обязательно", а "разрешить OCR при выполнении триггера":

- PDF-страница без текстового слоя, с коротким текстом или крупным изображением.
- Крупная картинка внутри DOCX/PPTX.
- Изображение прошло prefilter как документ, скан или скриншот.
- Видео обрабатывается только ограниченной выборкой кадров.

В CLI строка `OCR может понадоб.` показывает количество файлов с таким условным
OCR-шагом. Строка `OCR сразу` показывает планы, где OCR был бы обязательным
первым действием; в нормальном режиме она должна быть близка к нулю.

## Настройки

| Параметр | Что меняет |
|---|---|
| `planner.low_confidence_threshold` | Warning при низкой уверенности file type. |
| `planner.tiny_file_bytes` | Граница очень маленького файла. |
| `planner.small_image_bytes` | Когда маленькую картинку можно пропустить без OCR. |
| `planner.pdf_ocr_max_pages` | Максимум PDF-страниц для OCR escalation. |
| `planner.docx_ocr_max_images` | Максимум embedded images в DOCX. |
| `planner.presentation_ocr_max_images` | Максимум embedded images в презентации. |
| `planner.video_max_frames` | Максимум кадров видео для OCR. |
| `planner.suspicious_name_keywords` | Слова, делающие имя файла подозрительным. |
| `planner.business_context_keywords` | Слова, похожие на обычный бизнес-документ. |

Подробно: [settings guide](settings.md).

## CLI

```bash
pii-leak-detector scan share --plan
```

Команда сначала выполняет обычную инвентаризацию, затем печатает сводку планов извлечения.
