# Детектор ПДн `pii_detector.py`

## Назначение

`pii_detector.py` принимает `TextBlock`-и после базового извлечения текста и
возвращает факты обнаружения ПДн.

Этот модуль не решает, является ли файл утечкой. Он только отвечает на вопрос:
какие категории ПДн найдены, сколько их, где они найдены и с какой уверенностью.

## Контракты

### `PiiFinding`

Агрегированная находка одной категории в одном `TextBlock`.

Поля:

| Поле | Что означает |
|---|---|
| `file_path` | Исходный файл. |
| `category` | Категория ПДн. |
| `count` | Количество найденных значений. |
| `confidence` | Уверенность детектора. |
| `block_index` | Номер текстового блока. |
| `page_or_sheet` | Страница, лист, часть DOCX или диапазон строк. |
| `extraction_method` | Из какого extractor-а пришел текст. |
| `examples` | Маскированные примеры, без полных значений. |
| `detector` | Тип детектора: regex, checksum, keyword. |

### `PiiFileResult`

Агрегация находок по файлу:

- `total_count`;
- `categories`;
- `blocks_with_pii`;
- `findings`;
- `warnings`.

## Категории

Базовая версия ищет:

- `fio`;
- `email`;
- `phone`;
- `address`;
- `birth_date`;
- `passport_rf`;
- `snils`;
- `inn_person`;

- `biometric_data`;

- `political_views`;
- `nationality`.

## Валидация

Для снижения шума используются контрольные проверки:

- СНИЛС - контрольная сумма;
- ИНН физлица и юрлица - контрольные цифры;
- банковские карты - алгоритм Луна;
- паспорт РФ - контекстные слова рядом с номером;
- дата рождения, БИК, счет и CVV - контекстные слова рядом со значением.

ФИО пока определяется эвристически: три русских слова с заглавной буквы, где одно
похоже на отчество.

Специальные категории определяются keyword-детекторами. Это recall-friendly слой,
который должен потом уточняться `risk_classifier`.

## CLI

Запуск поверх базового извлечения:

```bash
python main.py share --detect-pii
```

Smoke-прогон:

```bash
python main.py share --detect-pii --extract-limit 180
```

Можно вывести все три сводки:

```bash
python main.py share --plan --extract --detect-pii --extract-limit 3500
```
