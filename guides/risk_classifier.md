# Risk classifier и submit writer

## Назначение

`risk_classifier.py` превращает найденные признаки ПДн в оценку подозрительности
файла и умеет формировать `.txt` submit для хакатонного Telegram-бота.

Этот слой использует:

- `ExtractionPlan` - формат, стратегия, контекст пути;
- `ExtractionRunResult` - удалось ли извлечь текст, какие шаги пропущены;
- `PiiFileResult` - категории и количества найденных ПДн.

## Контракты

### `RiskAssessment`

Поля:

| Поле | Что означает |
|---|---|
| `file_path` | Полный путь к файлу на диске. |
| `submit_path` | Путь от корня `share`, формат для бота. |
| `score` | Числовая оценка риска. |
| `level` | `low`, `review`, `submit`, `high`. |
| `categories` | Категории ПДн и количества. |
| `rule_hits` | Правила, которые повлияли на score. |
| `recommendation` | Рекомендация для ручной проверки. |
| `document_type` | Предполагаемый тип: таблица, профиль, список идентификаторов и т.п. |

### `RiskRuleHit`

Один вклад в score:

- `rule`;
- `score_delta`;
- `reason`.

## Скоринг

Скоринг сделан recall-friendly, потому что в условиях проверки ложный пропуск
подозрительного файла хуже лишнего файла в submit.

Сильные плюсы:

- банковские карты, CVV, MRZ;
- паспорт РФ, СНИЛС, ИНН физлица;
- массовые таблицы с ПДн;
- сочетание ФИО с паспортом/СНИЛС/ИНН/датой рождения/картой;
- неформальные employee-папки вроде `Мои бумажки`;
- пути с `выгруз`, `subscribers`, `dump`, `backup`, `анкета`, `заявка`.

Снижения:

- публичный/деловой контекст вроде `policy`, `rules`, `agreement`, `устав`,
  `отчет`, `disclosure`, `ukaz`, `fz`;
- бизнес-реквизиты организации без профиля физлица;
- похожие на публичный документ числовые совпадения без ФИО.

Дефолтный порог submit:

```text
120
```

Его можно менять флагом `--risk-threshold`.

## CLI

Показать risk-сводку:

```bash
python main.py share --risk
```

Сформировать submit:

```bash
python main.py share --submit out/submit.txt
```

Сформировать submit и debug report:

```bash
python main.py share --risk --submit out/submit.txt --risk-report out/risk_report.md
```

Порог можно менять:

```bash
python main.py share --submit out/submit.txt --risk-threshold 100
```

## Формат submit

Файл содержит только подозрительные пути, каждый на новой строке:

```text
/Прочее/file.pdf
/Выгрузки/дочерние предприятия/Billing/subscribers.csv
```

Путь считается от корня папки `share`, как требует бот.

## Текущий baseline

На текущем датасете `share` дефолтный порог `120` дает около сотни файлов в
submit. Это стартовая recall-friendly выдача для первой проверки ботом; после
получения F-score стоит тюнить порог и отдельные правила.
