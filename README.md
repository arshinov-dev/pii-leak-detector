# Tabular PII Detector

Детектор персональных данных (ПДн) в табличных файлах для интеграции в классификатор рисков утечек. Проект создан для хакатона и ориентирован на обнаружение российских типов ПДн.

## Возможности

- **6 типов обнаруживаемых ПДн:**
  - ИНН (10 и 12 знаков) — с математической проверкой контрольной суммы
  - СНИЛС (11 цифр) — с математической проверкой контрольной суммы
  - Паспортные данные (серия + номер)
  - Номера телефонов (российские форматы: +7 и 8)
  - ФИО — через NLP-модель Natasha
  - Адреса — через NLP-модель Natasha

- **Контекстный анализ** для снижения ложных срабатываний (проверка названий колонок и соседних ячеек)
- **Безопасное маскирование** найденных ПДн в отчёте (в логи не попадают реальные данные)
- **Многостраничные Excel-файлы** — обрабатываются все листы
- **Автоопределение кодировки CSV** (UTF-8 → CP1251)

Как это работает

Детектор проходит по каждой ячейке каждого листа таблицы и применяет четырёхступенчатый анализ.

Этап 1: Регулярные выражения

Поиск кандидатов по паттернам:

Тип	Регулярное выражение	Пример
ИНН	\b\d{10}\b или \b\d{12}\b	7714058912
СНИЛС	\d{3}[-\s]?\d{3}[-\s]?\d{3}[-\s]?\d{2}	123-456-789 01
Паспорт	\d{2}\s?\d{2}\s?\d{6} или \d{4}\s?\d{6}	4510 123456
Телефон	(?:\+7|8)[-\s]?\(?\d{3}\)?[-\s]?\d{3}[-\s]?\d{2}\b	+7(999)123-45-67
Этап 2: Математическая валидация

Для ИНН и СНИЛС проверяется контрольная сумма по официальным алгоритмам ФНС и ПФР.

ИНН (10 знаков — юрлицо):

text
Весовые коэффициенты: [2, 4, 10, 3, 5, 9, 4, 6, 8]
Контрольное число = (сумма произведений цифр на веса) % 11 % 10
ИНН (12 знаков — физлицо):

text
Два контрольных числа с разными весовыми коэффициентами:
w1 = [7, 2, 4, 10, 3, 5, 9, 4, 6, 8]
w2 = [3, 7, 2, 4, 10, 3, 5, 9, 4, 6, 8]
СНИЛС:

text
Сумма произведений первых 9 цифр на веса (9, 8, 7, ..., 1)
Если сумма < 100 → контрольное число = сумма
Если сумма 100 или 101 → контрольное число = 0
Иначе → сумма % 101 (если остаток 100 или 101 → 0)
Этап 3: Контекстный анализ

Проверяется название колонки и всё содержимое строки на маркерные слова:

Тип ПДн	Маркерные слова
ИНН	инн, налогопл, кпп, организац, огрн, контрагент, плательщик
СНИЛС	снилс, страхов, пенсион, сспс, физ.лицо, српф
Паспорт	паспорт, пасп, серия, номер, выдан, код, подразд, уд-ние, личности
Телефон	телефон, тел, сотов, мобил, номер, whatsapp, tg, связи, контакт, абонент
Логика:

Для паспортов и телефонов (где нет контрольной суммы) — наличие маркерного слова обязательно. Без контекста находка отбрасывается.
Для ИНН и СНИЛС — если контрольная сумма не сошлась, но колонка называется "ИНН" или "СНИЛС" — находка сохраняется с пониженной уверенностью (confidence = 0.65). Это позволяет ловить синтетические ИНН из тестовых сред.
Этап 4: NLP-анализ (Natasha)

Для текстовых ячеек длиной до 300 символов запускается модель Natasha:

Сегментация — разбивка текста на токены и предложения
NER-разметка — выделение именованных сущностей
Классификация:

PER (Person) → pii_type = "FIO"
LOC (Location) → pii_type = "ADDRESS"
Дополнительные фильтры:

Не анализируются ячейки, состоящие только из цифр
Не анализируются ячейки длиннее 300 символов (производительность)
Игнорируются колонки с названиями: дата, date, id, номер, индекс (чтобы не цеплять случайные токены)
Уровни уверенности (confidence)

Тип ПДн	Условие	Confidence
ИНН	Контрольная сумма сошлась	1.0
ИНН	Контрольная сумма НЕ сошлась, но есть контекст ("инн" в колонке)	0.65
СНИЛС	Контрольная сумма сошлась	1.0
СНИЛС	Контрольная сумма НЕ сошлась, но есть контекст	0.65
Паспорт	Найдено регуляркой + есть контекст	0.95
Телефон	Найдено регуляркой + есть контекст	0.85
ФИО	Найдено через Natasha PER	0.80
Адрес	Найдено через Natasha LOC	0.70
Как интерпретировать:

1.0 — математически подтверждённые ПДн (можно доверять полностью)
0.80–0.95 — высокая уверенность (есть и паттерн, и контекст)
0.65–0.70 — средняя уверенность (возможны ложные срабатывания)
Ниже 0.65 — находки не возвращаются
Маскирование данных

Все значения ПДн в выводе маскируются, чтобы не светить реальные данные в логах:

Тип ПДн	Правило маскирования	Пример
ИНН, СНИЛС, Паспорт	Первые 3 символа + *** + последние 2 символа	7714058912 → 771***12
Телефон	Первые 4 символа + *** + последние 3 символа	+79991234567 → +799***567
ФИО	Фамилия + первая буква имени с точкой	Иванов Иван Иванович → Иванов И.
Ограничения

Только российские ПДн — детектор не рассчитан на иностранные номера, паспорта и налоговые идентификаторы
ФИО и адреса — зависят от качества модели Natasha, возможны пропуски редких имён или адресов в нестандартном формате
Паспорта без контекста — если паспортные данные записаны в ячейке без маркерных слов в строке, они не будут обнаружены
Производительность на больших файлах — NLP-анализ (Natasha) работает относительно медленно. На файлах с десятками тысяч строк может занять несколько минут
Длина текста для NLP — ячейки длиннее 300 символов не анализируются через Natasha для сохранения производительности
Формат CSV — требуется, чтобы разделителем была запятая (стандартный CSV). Нестандартные разделители не поддерживаются
Телефоны — определяются только российские номера (начинающиеся на +7 или 8). Иностранные номера не детектируются
Объединённые ячейки в Excel — pandas может некорректно читать объединённые ячейки, что приведёт к пропуску данных
